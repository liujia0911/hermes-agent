## 二、8大职能 × 3个Agent组

| 职能 | 归属组 | Agent角色 | 核心动作 |

## 三、三个Agent群 × 每日工作流

### 3.1 内容策划组（佳哥IP + 睿哥IP）

**群成员：** 佳哥、睿哥、CEO Agent、内容Agent、设计Agent

**每日节奏：**

| 时间 | 动作 | 谁 |
|------|------|-----|
| 8:00 | 今日拍摄脚本推送到群 | @内容Agent |
| 8:30 | 今日宣传物料（海报+文案卡） | @设计Agent |
| 10:00 | 昨日内容数据（播放/互动/粉丝增长） | @数据Agent |
| 17:00 | 明日选题建议（基于今日数据） | @内容Agent |
| 周五17:00 | 下周拍摄排期表 | @内容Agent + @设计Agent |

**@触发示例：**
```
佳哥: "@内容Agent 明天拍什么"
  → Agent: "建议: 行业诊断·损失框架。选题: '你以为店租贵是问题，其实是没有线上入口'
           脚本已生成 → [链接]"

睿哥: "@设计Agent 销冠四条通道海报更新"
  → Agent: "已生成: 四条通道对比海报×1 + 竖版文案卡×4 → [文件]"
```

### 3.2 运营数据组

### 3.3 客户增长组

**群成员：** 佳哥、睿哥、CEO Agent、增长Agent、交付Agent

**每日节奏：**

| 时间 | 动作 | 谁 |
|------|------|-----|
| 9:00 | 昨日新线索汇总（评论'获客'收集的意向客户） | @增长Agent |
| 9:30 | 待跟进提醒（超24小时未联系的高意向） | @增长Agent |
| 10:00 | 跟进话术推送（根据客户行业定制） | @增长Agent |
| 11:00 | 新入驻客户欢迎 + 首周任务推送 | @交付Agent |
| 18:00 | 当日跟进汇总 + 意向分级更新 | @增长Agent |

**线索分级标准：**

| 级别 | 标准 | 跟进策略 |
|------|------|---------|
| S级 | 已留手机号+主动询问价格 | 30分钟内电话，当天发方案 |
| A级 | 评论'获客'+私信已回复 | 2小时内发产品介绍+案例 |
| B级 | 仅评论'获客'，未私信 | 自动私信+24小时内二次触达 |
| C级 | 点赞/关注，未互动 | 持续内容触达，不主动私信 |

**裂变追踪：**
```
@增长Agent 每日检测:
- 老客户是否在评论区推荐销冠
- 是否有"朋友介绍来的"新线索 → 标记推荐人 → 计入裂变返佣
- 合伙人/经纪人的下级线索是否在增长
```

## 四、每日宣传物料自动化

### 4.1 物料类型

| 类型 | 用途 | 频率 | 生成方式 |
|------|------|------|---------|
| 口播脚本 | 佳哥/睿哥拍摄 | 每日各1条 | @内容Agent |
| 封面海报 | 视频封面 | 每日各1张 | @设计Agent |
| 文案卡 | 评论区置顶/私信欢迎语 | 每日1套 | @设计Agent |
| 四条通道图 | 私信发送/主页展示 | 每周更新 | @设计Agent |
| 客户案例卡 | 朋友圈/私域分发 | 每周1-2个 | @设计Agent |
| 数据战报 | 社群/朋友圈信任背书 | 每周1张 | @数据Agent |

### 4.2 物料生成流程

```
1. @内容Agent 出脚本
2. @设计Agent 根据脚本自动生成:
   - 视频封面（钩子关键词+人物形象）
   - 评论置顶文案（3条变体）
   - 私信自动回复文案（含四条通道图）
3. 推送到内容策划组
4. 佳哥/睿哥确认
5. 按排期发布
```

## 五、客户对接SOP（Agent自动化部分）

### 5.1 评论'获客' → 自动私信

```
用户评论"获客"
  → @增长Agent 检测到
    → 自动私信（飞书/抖音私信）:
      "老板你好！我是睿哥团队的运营助手。
       销冠数字员工帮实体店做线上获客——四条通道同时跑。
       你想了解哪一块？回我数字就行：
       1-发视频获客  2-投流获客  3-搜索排名  4-AI推荐
       或者直接发你的行业+城市，我帮你看方案。"
```

### 5.2 意向客户跟进SOP

```
Day 0: 首次互动 → 发产品介绍 + 四条通道图
Day 1: 未回复 → 发行业案例（同行业老板的获客故事）
Day 3: 仍未回复 → 发限时福利/算力体验券
Day 7: 最后一触 → "老板还在考虑吗？有任何疑问随时问我"
Day 14: 归档 → 移入长期触达池，持续内容触达
```

### 5.3 成交客户交付SOP

```
成交当天:
  → @交付Agent: 发送入驻欢迎包（使用教程+后台地址+专属链接）
  → 创建客户档案（飞书Base）

第1天:
  → @交付Agent: "第一天感觉怎么样？素材准备好了随时发我"

第3天:
  → @交付Agent: 首次数据播报（已发视频数+播放量）

第7天:
  → @交付Agent: 首周数据报告 + 优化建议

之后:
  → 每周一自动播报上周数据
  → 异常预警（连续3天数据下滑）
  → 续费提醒（到期前7天）
```

## 六、裂变机制

### 6.1 裂变路径

```
老客户 → 推荐新客户 → 新客户成交 → 老客户获返佣
合伙人/经纪人 → 发展下级客户 → 下级消耗算力 → 合伙人获分成
```

### 6.2 Agent追踪点

| 节点 | Agent检测 | 动作 |
|------|----------|------|
| 新线索来源标记"朋友推荐" | @增长Agent | 追溯推荐人，计入裂变记录 |
| 推荐人本月推荐数达到阈值 | @增长Agent | @CEO Agent，触发额外激励 |
| 合伙人下级客户算力消耗 | @数据Agent | 计算分成，生成结算单 |
| 裂变转化率持续低于基准 | @增长Agent | 预警，建议优化裂变激励机制 |
