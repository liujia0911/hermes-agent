## 场景一：销冠数字员工 — 客户交付全链路

### 现状痛点
- 2人团队，每次新客户交付链路长：上门拍摄 → 系统配置 → 训练部署 → 官网搭建 → 培训客户
- 步骤靠人记，容易遗漏
- 每次都要翁锐手动操作

### Skill组合方案

```
一、skill-factory 沉淀标准SOP
    ↓
二、maestro 每次新客户自动拆解+调度
    ↓
三、cronjob 交付后定时回访提醒
```

**具体操作：**

```bash
# 1. 用 skill-factory 自动生成交付SOP
/skill-factory --from-command "销冠客户交付全流程：
1. 收到客户资料 → 创建飞书多维表格登记
2. 佳哥上门拍摄 → 素材上传到指定飞书文件夹
3. 数字员工配置 → 开通账号+配置5项消耗参数
4. 训练部署 → 上传素材+训练模型+测试生成
5. 官网搭建 → 创建官网+配置SEO关键词
6. 培训客户 → 发操作手册+视频教程+1v1答疑
7. 3天后回访 → 飞书消息提醒
8. 7天后复盘 → 检查消耗数据+客询转化"
```

**预期产出：** `delivery-sop` Skill — 每次说"新客户交付"自动走8步清单，不遗漏。

```bash
# 2. 用 maestro 一次搞定交付
/maestro 新客户交付：客户名称XX公司，行业餐饮，
  佳哥已拍摄完毕，素材在飞书文件夹"XX公司素材"，
  需要完成配置、训练、官网、培训全流程
```

**预期产出：** maestro自动拆解成3-5个子任务，并行执行配置+官网搭建，串行执行训练→测试→培训。

## 场景二：乌贼帮 — 月度内容生产管线

### 现状痛点
- 三平台（抖音/视频号/小红书）同步运营
- 月8条视频+社群运营+线索收集
- 内容策划→拍摄→剪辑→发布→数据复盘，全靠翁锐手工推进

### Skill组合方案

```
一、cronjob 设定时任务链
    ↓
二、maestro 月初策划拆解
    ↓
三、skill-factory 沉淀内容生产SOP
```

**具体操作：**

```bash
# 每周一 9:00 — Codex扫描抖音热点+生成5个选题
/cronjob create --name "乌贼帮-周一选题" \
  --schedule "0 9 * * 1" \
  --prompt "用Codex扫描抖音美食/摆摊/加盟类热点，结合乌贼帮潮汕炒乌贼主题，产出5个视频选题"

# 每周二 18:00 — 自动复盘上周数据
/cronjob create --name "乌贼帮-数据周报" \
  --schedule "0 18 * * 2" \
  --prompt "从飞书多维表格读取乌贼帮上周数据，生成数据周报发送到飞书群。"

# 每天 20:00 — 检查当日发布状态
/cronjob create --name "乌贼帮-发布检查" \
  --schedule "0 20 * * *" \
  --prompt "检查今天乌贼帮三个平台是否都已发布，未发布的提醒翁锐。"
```

```bash
# 月初用maestro做整月策划
/maestro 乌贼帮6月内容策划：
  目标：8条视频 + 50条线索 + 5000粉丝净增
  平台：抖音主攻、视频号同步、小红书图文
  角色：QS版乌贼（萌）、泳哥（硬核拆解）、英歌舞（文化）
```

## 场景三：招商策划 — 多角度评审

### 现状痛点
- 招商方案是佳哥+翁锐两个人拍脑袋
- 没有系统性的成本/风险/话术/合规审查

### Skill组合方案

```
gstack 做多角色评审
  ├─ CEO角色：目标对齐（3个月100万）
  ├─ Security角色：合规审查
  ├─ Architect角色：经纪人体系架构
  └─ QA角色：边界条件测试
```

```bash
/gstack --skip designer,perf,devops,release \
  销冠数字员工经纪人招商方案：成本结构见招商文档，
  目标3个月100万净利润，用经纪人+城市合伙人模式。
  从CEO/合规/体系/风险四个角度审查。
```

## 关键原则

1. **先跑起来再优化。** P0动作今天就开始，不追求一步到位。
2. **定时任务是你的第三个员工。** 机器不会忘、不会累、不会请假。
3. **Skill是资产不是负担。** 每沉淀一个SOP，就解放你一点时间。
4. **maestro解决"复杂任务靠人记"的问题。** 2人团队最缺项目管理。
5. **gstack在重大决策前用。** 日常不用，杀鸡不用牛刀。
