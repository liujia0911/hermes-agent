# 销冠抖音运营部 · Agent角色能力矩阵

> 2026年6月 | 昕冠科技
> 所有Agent由Hermes担任，通过不同模式/工具集/技能组合区分角色

---

## 一、角色总览

| Agent | 谁担任 | 实现方式 | 核心能力需求 |
|-------|--------|---------|------------|
| CEO Agent | Hermes（默认模式） | 主会话/cron | 全局调度、决策分析、跨Agent协调 |
| 内容Agent | Hermes（内容模式） | delegate_task | 脚本生产、选题策划、文案审查 |
| 设计Agent | Hermes（设计模式） | delegate_task | 海报/封面/物料生成 |
| 运营Agent | Hermes（运营模式） | cronjob | 账号策略、竞品分析、规则追踪 |
| 数据Agent | Hermes（数据模式） | cronjob | 数据抓取、分析、可视化、预警 |
| 增长Agent | Hermes（增长模式） | cronjob | 线索管理、跟进SOP、裂变追踪 |
| 交付Agent | Hermes（交付模式） | cronjob | 客户入驻、教程推送、回访 |

## 二、逐角色能力拆解

### CEO Agent — 指挥调度

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes主会话（就是现在的我） |
| 运行方式 | 主会话 + 指挥中心群@触发 |
| 核心职责 | 接收佳哥/睿哥指令，分解任务分发给各Agent，汇总结果，做最终决策 |
| 需要工具 | 全部工具（terminal/file/web/skills/cronjob/delegate_task/send_message） |
| 需要技能 | sales-champion-marketing、douyin-b2b-ad-scripts、lark-im |
| 触发时机 | 佳哥/睿哥@小马下达指令；Agent升级求助；每日汇总 |

**能力清单：**
- [x] delegate_task：把任务分发给其他Agent
- [x] cronjob：管理定时任务（数据播报、内容排期）
- [x] send_message：飞书群通信
- [x] 全部skill：掌握公司所有方法论和业务知识
- [x] 决策判断：什么该自己做、什么该分发、什么该请示人类


### 内容Agent — 脚本和选题

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes delegate_task 子Agent |
| 运行方式 | CEO Agent调 delegate_task(content_mode) 或 每天8点cronjob |
| 核心职责 | 出拍摄脚本、选题策划、口播逐字稿、文案审查 |
| 需要工具 | file（读写脚本）、terminal（文件操作） |
| 需要技能 | sales-champion-marketing、douyin-b2b-ad-scripts、humanizer |
| 触发时机 | 每天8:00自动出当日脚本；佳哥/睿哥@内容Agent即时出 |

**能力清单：**
- [x] 掌握双轨话术（增益×损失框架）
- [x] 黄金3秒钩子设计
- [x] Humanizer去AI味口播
- [x] 选题维度矩阵（行业诊断/认知颠覆/趋势预判/案例拆解/数据披露）
- [x] 禁忌词汇过滤（零"干货""教程""一键"）
- [x] 佳哥风格 vs 睿哥风格切换
- [ ] 热点追踪（可选升级）：抓取当日抖音热点，融合进选题


### 设计Agent — 海报和物料

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes delegate_task 子Agent |
| 运行方式 | 内容Agent出脚本后自动触发，或@设计Agent即时生成 |
| 核心职责 | 视频封面、文案卡片、四条通道海报、数据战报图 |
| 需要工具 | image_generate（AI生图）、file、terminal |
| 需要技能 | frontend-design（HTML生成封面）、high-end-visual-design、infographic（可选） |
| 触发时机 | 每天8:30自动出当日物料；佳哥/睿哥@设计Agent即时出 |

**能力清单：**
- [x] AI生成视频封面（钩子关键词+人物形象模板）
- [x] 竖版文案卡（一段金句+品牌标识）
- [x] 四条通道对比图（发视频/投流/SEO/GEO）
- [x] 统一品牌视觉（销冠LOGO+配色体系）
- [ ] 品牌VI体系（可选升级）：定义销冠数字员工标准色板/字体/模板
- [ ] 视频字幕模板（可选升级）：设计Agent生成的字幕样式


### 运营Agent — 策略和规则

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes cronjob（每天9:30运行） |
| 运行方式 | 定时运行 + @运营Agent即时触发 |
| 核心职责 | 发布策略建议、平台规则追踪、竞品监测 |
| 需要工具 | web_search、browser（看竞品主页）、file |
| 需要技能 | sales-champion-marketing |
| 触发时机 | 每天9:30策略建议；平台规则变动时预警；@运营Agent即时分析 |

**能力清单：**
- [x] 分析抖音算法趋势和内容偏好
- [x] 追踪竞品IP账号的更新频率/内容方向/数据表现
- [x] 建议最佳发布时间和话题标签
- [x] 平台新规预警（如限流规则、内容审核标准变化）
- [ ] 对标账号数据库（可选升级）：维护竞品IP数据，做月度对比


### 数据Agent — 抓取和分析

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes cronjob（每天9:00运行） |
| 运行方式 | 定时运行，抓取昨日数据，生成播报 |
| 核心职责 | 数据抓取、分析、可视化、异常预警、AB测试 |
| 需要工具 | terminal（脚本抓数据）、file、browser（登录后台） |
| 需要技能 | 无专属skill，用Python脚本+浏览器自动化 |
| 触发时机 | 每天9:00自动播报；异常时实时预警 |

**能力清单：**
- [x] 每日抓取双账号播放/互动/粉丝数据
- [x] 生成数据播报卡片（对比昨日、7日均值）
- [x] 异常预警：播放暴跌30%+ / 互动率跌破基准 / 掉粉异常
- [x] 周报自动汇总生成
- [ ] 实时数据接入（当前为手动/定时抓取，可用抖音开放平台API升级）
- [ ] AB测试追踪（同一选题不同封面/发布时间的效果对比）

**当前数据源：** 浏览器登录抖音创作者后台，或用抖音开放平台API（需佳哥/睿哥授权）。

**能力清单：**
- [x] 评论"获客"自动检测（抖音创作者后台评论管理）
- [x] 线索分级（S/A/B/C四级）

- [ ] 抖音私信自动化接口（当前需人工，可用抖音企业号API升级）

### 交付Agent — 入驻和回访

| 维度 | 内容 |
|------|------|
| 担任者 | Hermes cronjob（每天11:00+18:00） |

| 核心职责 | 新客入驻引导、使用教程、首周回访、续费提醒 |
| 需要工具 | lark-im（飞书消息）、lark-doc（出教程）、file |

**能力清单：**
- [x] 自动推送入驻欢迎包（后台地址+使用教程+联系人）
- [x] 创建客户档案（飞书Base录入）
- [x] D+1/D+3/D+7回访提醒和消息推送
- [x] 首周数据报告自动生成
- [x] 续费前7天提醒
- [x] 异常使用预警（客户3天未登录后台→主动关心）
- [ ] 客户健康评分（可选升级）：根据使用频率/数据增长/续费概率综合评分

## 三、能力缺口 × 优先级

| 缺口 | Agent | 优先级 | 方案 |
|------|-------|--------|------|
| 抖音数据自动抓取 | 数据Agent | P0 | 浏览器自动化登录创作者后台，每天抓数据 |
| 抖音评论/私信轮询 | 增长Agent | P0 | 浏览器自动化或手动检查，先人工确认再自动 |
| 设计Agent品牌模板 | 设计Agent | P1 | 用frontend-design或image_generate建立模板库 |
| 竞品数据库 | 运营Agent | P1 | 手动建立5-10个对标账号，每月更新 |
| 抖音开放平台API | 数据+增长 | P2 | 佳哥/睿哥在抖音开放平台创建应用授权 |
| 客户健康评分 | 交付Agent | P2 | 先用人工判断，数据积累后做模型 |
