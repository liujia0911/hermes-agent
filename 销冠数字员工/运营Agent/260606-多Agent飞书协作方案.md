## 二、Agent角色定义

| Agent | 飞书Bot | 核心职责 | 工作群 |
|-------|---------|---------|--------|
| CEO Agent | @小马 (现有) | 全局调度、决策审批、日报汇总、战略分析 | 全部群 |
| 内容Agent | @小马(内容模式) | IP脚本生产、选题策划、文案审查 | 内容工坊 |
| 投流Agent | @小马(投流模式) | 广告搭建、数据监控、预算优化 | 投流作战室 |
| 交付Agent | @小马(交付模式) | 客户视频产出追踪、周报生成、异常预警 | 客户交付中心 |

**实现方式：** 所有Agent共享同一Hermes实例，通过cronjob+delegate_task区分角色。对外表现为不同Bot，群内@不同角色触发不同任务。

## 三、群组通信协议

### 3.1 @触发规则

在指挥中心群里，任何人@Agent会触发对应任务：

| @谁 | 示例 | Agent做什么 |
|-----|------|------------|
| @CEO Agent | "@小马 汇总本周各客户数据" | 调各客户交付群数据，生成周报 |
| @内容Agent | "@小马 佳哥明天拍什么" | 检查内容日历，输出明日拍摄脚本 |
| @投流Agent | "@小马 崃州投流数据" | 拉取巨量AD昨日数据，输出简报 |
| @交付Agent | "@小马 乌贼帮本周进度" | 汇总乌贼帮视频产出和线索数据 |

### 3.2 Agent自主@Agent

Agent之间可以互相@触发协作链：

```
投流Agent检测到CTR下降
  → @内容Agent: "脚本3的CTR跌破2%，建议换钩子"
    → 内容Agent出新版脚本
      → 投流Agent更新素材
        → @CEO Agent: "崃州脚本3已优化上新"
```

### 3.3 定时自动播报

| 时间 | 播报内容 | 播报群 |
|------|---------|--------|
| 每天9:00 | AI生态扫描日报 | 指挥中心 |
| 每天10:00 | 昨日投流数据汇总 | 投流作战室 |
| 每周一9:00 | 上周客户产出周报 | 客户交付中心 |
| 每周五17:00 | 下周内容拍摄计划 | 内容工坊 |

## 四、任务流转机制

### 4.1 任务卡片格式

Agent之间用统一的任务卡片格式通信：

```
📋 任务 #20260607-001
发起: @投流Agent
指派: @内容Agent
内容: 崃州脚本3 CTR从3.2%跌至1.8%，需更换钩子方向
参考: 当前数据截图
截至: 今日18:00
状态: 待处理
```

### 4.2 状态流转

```
待处理 → 处理中 → 已完成/已驳回
         ↓
      超时预警 → @CEO Agent升级
```

### 4.3 客户专项群协议

每个客户一个飞书群（如"乌贼帮·运营群"），群内有：
- 佳哥/睿哥（人类）
- 客户（人类）
- 交付Agent
- CEO Agent

交付Agent每天在群里自动播报：
- 今日已发布视频链接
- 昨日播放量/互动量
- 本周线索数 vs KPI
- 异常预警（播放量骤降等）

## 五、Agent自主开会流程

### 5.1 触发条件

- 每周复盘会：周一下午3点，交付Agent自动创建飞书会议
- 异常升级会：投流数据或客户数据出现异常时，CEO Agent判断是否需要拉会
- 客户复盘会：客户项目到期或里程碑节点，交付Agent发起

### 5.2 会议自动化流程

```
1. Agent创建会议 → 拉入相关人员
2. Agent加入会议 → 实时录音转文字
3. 会议结束 → 自动生成纪要(MOM)
4. 提取待办 → 创建飞书任务分配
5. 发送群消息: "会议已结束，3条待办已分配，纪要链接: [URL]"
6. 待办到期前1小时 → @提醒负责人
```

### 5.3 会议类型

| 类型 | 频率 | 参与者 | Agent角色 |
|------|------|--------|----------|
| 晨会(站会) | 每日9:15 | 佳哥+睿哥+CEO Agent | Agent播报昨日数据，提今日重点 |
| 周复盘 | 每周一 | 全员 | Agent出周报，人审+讨论 |
| 客户复盘 | 每月 | 客户+佳哥/睿哥+Agent | Agent出月报，人跟客户沟通 |
| 异常会 | 按需 | 相关人员 | Agent检测异常→CEO判断→拉会 |

## 六、数据流与看板

### 6.1 数据汇聚

```
各客户交付群(原始数据)
    → 交付Agent汇总
      → 飞书Base多维表格(结构化存储)
        → CEO Agent出分析报告
          → 发指挥中心群
```

### 6.2 核心看板（飞书Base）

| 表格 | 字段 | 更新频率 |
|------|------|---------|
| 客户项目表 | 客户名/服务类型/月费/开始日期/KPI/当前状态 | 实时 |
| 内容产出表 | 客户/平台/视频标题/发布时间/播放量/互动量 | 日更新 |
| 投流数据表 | 项目/广告组/消耗/CTR/CPL/线索数 | 日更新 |
| 任务追踪表 | 任务ID/发起人/负责人/内容/状态/截止日 | 实时 |
| 招商线索表 | 来源/联系方式/行业/跟进状态 | 实时 |

## 八、风险与边界

- Agent不替代人做最终决策。投流预算调整、客户沟通、内容方向——人类审批后再执行。
- 客户群里的Agent发言需经CEO Agent审核。不发未审核的内容。
- Agent之间的@通信有频率限制（飞书API限流）。批量任务用cronjob而非实时@。
- 首次部署用乌贼帮项目试跑，跑通后复制到其他客户。
